# CS-320-Software-Test-Automation-QA
Collection of CS 320 coursework demonstrating software testing strategies, automated test development, and QA processes.

How can I ensure that my code, program, or software is functional and secure?
  I ensure functionality and security by combining thorough testing with strong input validation. In the Contact Service project, I wrote unit tests to verify that each method behaved as expected, including edge cases like invalid IDs or overly long fields. I also enforced constraints directly in the code, such as limiting character lengths and preventing null values. This approach helps catch errors early and protects the system from invalid or potentially harmful input.

How do I interpret user needs and incorporate them into a program?
  I interpret user needs by analyzing requirements and translating them into specific features and constraints. For example, in the Contact Service, user needs included storing contact information and ensuring each contact had a unique ID. I implemented this by designing classes that enforce uniqueness and validate fields like name, phone number, and address. By breaking requirements into smaller, testable components, I can ensure the program aligns with what the user expects.

How do I approach designing software?
  I approach software design by starting with a clear structure and building modular, maintainable components. In the Contact Service project, I separated responsibilities across classes, such as a Contact class for data and a service class for managing contacts. I focused on simplicity, readability, and scalability, making sure each part of the program had a single responsibility. I also considered how the system might grow, designing it in a way that allows for easy updates or additional features later. For the project conclusion, I upgraded these systems to make them more comprehensive and extend their capabilities.
